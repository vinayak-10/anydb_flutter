# Session: 2026-06-23-1

## Summary
Focused on diagnosing and resolving the "Empty Monthly Report" issue and clarifying the "Date Picker" and "Documents Hierarchy" behaviors.

## Findings

### 1. Empty Monthly Report (In-App Display)
- **Root Cause**: The `ExtractorReport._prepare` method in `lib/services/extractor_service.dart` uses hardcoded indices (Row 5 for headers, Row 6 for values) to extract data from daily sheets when no cell coordinate is provided.
- **Failure**: Since generated sheets often have different row offsets (e.g., table data starting at Row 8), the extractor fails to find matching titles, returns 0 or empty values, and results in an empty `aoa` (Array of Arrays) being passed to the UI.
- **Agreed Fix**: Implement a dynamic header search in `ExtractorReport._prepare` that scans the sheet for the target title and extracts the value from the immediately following row.

### 2. Date Picker UI
- **Observation**: The UI uses `DateFormat('MMMM yyyy')` for monthly reports.
- **User Clarification**: This is expected behavior for the display. The perceived "bug" was actually a symptom of the missing report data (point 1).

### 3. Documents Directory Hierarchy
- **Finding**: `WorkbookService._backupDatabase` performs a shallow scan.
- **User Direction**: Handling the directory hierarchy inside `WorkbookService` is considered the wrong approach. Do not implement fixes there.

## Next Steps
- Implement the dynamic header search in `lib/services/extractor_service.dart` to fix the empty monthly report display.
